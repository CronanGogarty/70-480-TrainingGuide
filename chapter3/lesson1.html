Understanding JavaScript

JavaScript is used to add interactivity to web pages. It can be used to create animations, add functionality when a user-event such as a button-click occurs, or to retrieve data from a web server when a user enters text into a form-field.

JavaScript is related to ECMAScript. ECMAScript is the standardized language for client-side scripting on the web and some variations on ECMAScript include JavaScript, JScript and ActionScript.

JavaScript is an untyped language, this means that when you declare a variable, you do not need to specify it's type.

Understanding Data

Data is at the root of all systems. When you create a program, you will typically want to retrieve, manipulate, display and store data.

Data can be stored in different forms but most data is broken into small pieces called values. In JavaScript a value type is a primitive value, a function or an object,. In JavaScript a primitive value is one of the following;

  • undefined
  • null
  • Boolean
  • number
  • string

A function is a callable object. Afunction that is part of an object is known as a method.

JavaScript defines the following built-in objects;

  • the global object
  • the Object object
  • the Function object
  • the Array object
  • the String object
  • the Number object
  • the Math object
  • the Date object
  • the RegExp object
  • the JSON object
  • several variations of Error objects

Using expressions to produce data

An expression is a piece of code that produces a values such as a simple declaration of a value or a calculation using operators and operands. The operators are typically the mathematical symbols within an expression (+, -, *, etc...) while the operands are the values either side of the operators. Operands are typically numbers or strings but can also be other expressions.

Understanding the number type

In JavaScript the number type is one of the primitive value types. A numeric value is a member of the number type and corresponds to a double-precision, 64-bit binary format. In JavaScript all numbers are represented internally as floating-point values. The 64-bit number is divided into 3 components: the fraction is 52 bits (bits 0-to-51), the exponent is 11 bits (bits 52-to-62) and the sign is a single bit (bit 63).

The highest number that can be represented internally is 253 which is 9,007,199,254,740,994. After that numbers are stored as a fraction times 2exponent.

When performing integer calculations the results will always be precise, but when dealing with fractions the results are often incorrect due to the manner in which fractions are stored. It is best to round your fractions to a specific number of decimal places.

The number type supports some special values;

  • NaN - not a number
  • Infinity - Represents positive infiity when your value exceeds 1.7976931348623157E + 10308
  • -Infinity - Represents negative infiity when your value exceeds -1.7976931348623157E + 10308.
  • undefined - no value has been assigned
Operator Precedence

Operator precedence determines the order in which operators are evaluated. Operators with higher precedence are evaluated first. The plus (+) and minus (-) signs have lower precedence than the multiplication (*) and division (/) signs which are equal to the modulo (%) or remainder sign. Precedence can be forced by using parentheses to determine which expressions are to be evaluated first.

Modulo

The modulo (%) operator performs an implied division and returns the remainder. For example, 25 % 7 produces a result of 4, which is the remainder after dividing 25 by 7.

Understanding the string type

A string is a collection of characters and is used to represent text. Strings can be enclosed in double or single quotes, it doesn't matter so long as the opening and closing quotes match.

The backslash character is used to escape special characters and place them in a string - e.g. "This string has a \"double quotes\" escaped in it.". Other escape sequences include \t for a tab, \n for the newline character and \u0233 will embed the unicode symbol é in a JavaScript string.

Using the plus (+) operator on strings performs string concatenation - e.g. "This string " + "has been concatenated!" will create a new string with the value "This string has been concatenated!".

Unary Operators

Most operators are binary, in that they require two operands. When an operator requires only one operand it is known as a unary operator. The typeof operator is an example of a unary operator, it takes one operand and returns the dat type of that operand - e.g. typeof "Any text." will return a value of 'string', typeof 78.9 will return 'number' and typeof false will return 'Boolean'.

The plus and minus signs can be used as unary operators to specify the sign of numbers - e.g. -35. The not operator (!) is also a unary operator.

The Boolean type

The Boolean type can contain the values true and false.

Logical Operators

JavaScript provides three logical operators that can be used in expressions to produce Boolean values;

  • && - logical and - true if both values evaluate to true
  • || - logical or - true if either values evaluate to true
  • ! - the not operator - a unary operator that will invert the operand - e.g. !false is true
Short-circuiting operators

In the case of the logical or (||) operator, if the first operand is true, JavaScript will not evaluate the expression any further as the result of the expression will evaluate to true. Similarly, if the first operand of a logical and (&&) expression is false JavaScript will abandon the rest of the expression as the result (false) is already known.

Using Statements

In JavaScript a statement is a line of code that ends in a semicolon (;). A statement is different from an expression because an expression produces a value then JavaScript discards it. An statement tells the JavaScript host what to do. The host is typically a browser but could also be an operating system. A statement can be a command to store the value produced by an expression.

The results of expressions are typically stored in variables. A variable is a named reference to a location in memory for storing data. Variables in JavaScript are created using the var keyword.

You declare your variables when you write code that states "here is a variable named myVar" = var myvar;. You initialize that variable when you give it a value = myvar = 123.25;.

Rules for naming variables

  • variable names can contain letters, numbers, the underscore (_) symbol and the dollar ($) sign
  • a variable name cannot begin with a number
  • no mathematical or logical operators or punctuation symbols are allowed in a variable name
  • variable names cannot be JavaScript keywords or reserved words - keywords and reserved words are allowed within the variable name - e.g. var function; is illegal - var myfunction; is legal
  • no spaces are allowed
  • variable names are case-sensitive
  • although allowed, it is recommended not to use $ or _ in variable names

Creating the environment

The collection of all variables and their values is commonly referred to as the environment. When the environment is created, it contains many standard variables plus the variables you create. In a web application, each time a webpage is loaded into the browser, a new environment is created, and the old environment is destroyed. Any variables that you create are accessible until a new webpage is loaded. In a Windows 8 program, an environment is created when the application starts, and the environment is destroyed when the application ends. A variable is accessible as long as your program is running.

Working with functions

A function is a grouping of statements that are executed when you call the function. Functions promote code reuse because you can call the function many times from within your code. Functions can have parameters, which enable you to pass data into the function. Functions can also have a return value, so you can return the results of the function to the caller.

A function can be declared by using the function keyword and then providing a name (also known as an identifier), the optional list of parameters enclosed in parentheses, and a set of curly braces with the grouping of statements. e.g 

function multiplyNumbers(n1, n2)    {
    return n1 * n2;
}

This is an example of a function declaration. The name (or identifier) is multiplyNumbers, it takes two parameters (n1, n2) and has a code block denoted by the curly braces.

You can call the function from code by writing var result = multiplyNumbers(3,7); - this is also known as invoking or applying the function. In the given example, 3 and 7 are the arguments to the multiplyNumbers function. Arguments are what's passed to the receiving function in the code that calls it (3 and 7), while parameters are the values received from the caller (n1 and n2). The variable "result" will hold the value returned by the function. Not all functions will return a value.
