Adding images to your HTML documents

The <img> tag is a void tag, that is it does not contain any content. Img has two required attributes;

  • src - an absolute or relative URL reference to the image that is to be displayed
  • alt - provides alternate text to be displayed when the image is not available

It is important to realise that the image is not embedded in the HTML document, it is referenced. When the browser encounters an image tag, it will retrieve the referenced image by folowing the value in the src attribute and merge the retrieved image into the final rendering that is displayed in the browser window.

Image Types

When using the img element it is posdible to use the following file types;

  • .JPG - Also known as JPEG, this is best for photographs because it offers high compression and up to 16.8 million color combinations, but the compression algorithm is lossy, meaning that you lose detail every time you save the fie.
  • .GIF - GIF is great to use on small images that have a fied number of colors. GIF also supports transparent color. GIF uses lossless compression and is best for logos and worst for photos. GIF also supports the ability to encapsulate multiple images in one fie, which is commonly used to provide animated GIFs.
  • .PNG - PNG is a great all-around fie type due to its lossless high compression. PNG fies can be 48-bit true color or 16-bit grayscale. PNG not only supports transparent color but also offers variable transparency. Photos aren’t compressed to be as small as JPG photos, but being lossless makes it worth the extra size in many scenarios.
  • .SVG - SVG is Scalable Vector Graphics and is great for drawings but not for photos. SVG images can be scaled up or down without losing detail because the fie contains the instructions to draw the image, as opposed to the other fie types that contain a raster-based image. Raster-based images are composed of color dots that make up the image. If you need to scale a raster-based image up or down, you will see that in color, dots are re-sampled, and the image typically ends up looking blocky.

Image maps

It is possible to create a map which can then be associated with an image using the usemap="mapName" attribute of the img element. The map element requires a name="mapName" attribute, this is the name that is then used when setting the usemap attribute for the img element. In the map element you set area elements which have an attribute of coord which can take the following values;

  • rect - x1, y1, x2, and y2 specify the coordinates of the left, top, right, and bottom
  • circle - x, y, and radius specify the coordinates of the center and the radius
  • poly - x1, y1, x2, y2,.., xn, and yn specify the coordinates of the edges. The fist and last coordinate pairs should be the same to close the polygon, but if they aren’t the same, the browser will add a closing pair.

Each area element can have a href attribute and a alt attribute which are similar to those in the a href element.

Embedding plug-in content

There are two supported elements for embedding plug-in content;

  • <embed>
  • <object>

<embed> was originally created by Netscape and then support for the element was added to Internet Explorer, the <object> element was added to the HTML4.0 specification. In HTML5 both elements are valid - object provides more functionality whereas embed is easier to use.

The embed tag has the following attributes;

  • height
  • width
  • src
  • type - specifies the MIME type of the embedded content

For browsers that do not support the embed element, fallback code can be added between the opening and closing tags;
<embed src="myFlash.swf" height="100" width="300" type="application/x-shockwave-flash">Your browser does not support Flash - you are missing out</embed>

The object tag embeds an object, such as multimedia like audio, video, Java applets, ActiveX, PDF and Flash, within an HTML document. The object tag supports the following attributes;

  • data - Supplies the URL of the resource to be used by the object
  • form - Indicates one or more form ids to which the object belongs
  • height
  • width
  • name
  • type - Defies the MIME type of data specifid in the data attribute
  • usemap - the name of a client-side image map to be used with the object

At a minimum, you must define either the data or type attribute.

You can use the object element to embed another HTML page within your document as follows;
<object data="menu.html" type="text/html" name="embeddedMenu" height="100" width="400">
Alternate text to be displayed by browsers that do not support the object tag
</object>
using the iframe tag is the recommended approach for embedding a webpage since you can use the sandbox attribute to control content from an external domain. In addition, some features such as tooltips might not behave as expected when using the object tag.

Passing parameters to an object

Use the <param> tag to pass data to an object - these tags must be before any alternate text between the opening and closing tags. The param tag requires both name and value attributes. Here is an example;
<object data="someMusic.wav">
<param name="autoplay" value="false" />
Alternate text for non-supporting browsers
</object>

Lesson Summary

  • You can use the <iframe> element to provide reuse of HTML by embedding an HTML document in your current HTML document.
  • Use the sandbox attribute on the <iframe> tag to help prevent malware and annoyances such as pop-ups from being introduced when the content is embedded in your HTML page.
  • Use the seamless attribute on the <iframe> tag to indicate that the source content is to be rendered to appear as though it’s part of the containing document.
  • The <a> tag creates a hyperlink to either an external HTML document or an internal location in the current document. The <a> tag can also be used to send email messages.
  • The <img> element is a void element and is used to add an image reference to your HTML document.
  • JPG is best for displaying photos on HTML pages due to its compression, GIF is best for small images with transparency and embedded animations, PNG is best for storage due to lossless compression during editing sessions, and SVG is best for drawings due to its vector drawing scalability.
  • You can create a clickable image map by using the <map> and <area> elements.
  • You can use the <embed> tag to provide simple content embedding.
  • You can use the <object> tag to provide content embedding with greater flexibility because it can have nested <param> elements.
