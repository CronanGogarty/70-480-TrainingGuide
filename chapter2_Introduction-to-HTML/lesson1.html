Understanding HTML XHTML and HTML5

HTML

HTML is a descended from SGML (Standard Generalized Markup Language) which was used to document publishing.

HTML browsers were designed to be backwards-comatible and forwards-comatible. This was achieved by the browsers ignoring any tags that were not recognized, this means that older browsers simply ignore tags that were introduced since the browser was developed and released.

XHTML

XHTML is an XML-based specifiation that tightened the HTML specifiation to make HTML adhere to the XML rules that describe a well-formed document, such as having a matching end tag for each starting tag. This meant that XHTML documents could be validated by using XSD (XML Schema Definition) fies and could be edited by using XML tools

XHTML required that elements be defined in lowercase. All elements must be closed; in the case of empty and void elements this was achieved using self-closing tags.

The demand for additional multimedia and interactivity saw the introduction of various 3rd-party plugins and scripting languages such as JavaScript. There was need for tighter integration of of multimedia with the browsers (especially on mobile or smaller devices) and standardization across the spectrum of web technologies.

HTML5

HTML5 was developed by the W3C to provide the standardization and integration required by the demand for interactivity and multimedia elements across a wide variety of browsers and devices.

HTML5 originates from HTML 4.01 not XHTML!

"HTML5 represents a reinvented HTML, CSS, and JavaScript in a way that solves the need for rich, interactive websites that can play audio and video and support animations from within the browser without the need for plug-ins."

Introducing Semantic Markup

HTML5 stresses seperation of;

  • Structure
  • Presentation
  • Behaviour

HTML tags provide structure - the contents of the document are contained and described using HTML tags.
CSS defines the presentation of that structure - the same structure can be prsented in a different manner across user-agents.
Interactivity is provided by JavaScript - JavaScript is used to respond to user actions and other events.

Remember that HTML only provides structure and not presentation; even a tag such as <b> is defined by the W3C as "a span of text offset from its surrounding content without conveying any extra emphasis or importance, and for which the conventional typographic presentation is bold text; for example, keywords in a document abstract, or product names in a review."

Working with Attributes

The beginning tag of an HTML element can contain additional data in the form of an attribute. An attribute is a name-value pair where the name is unique within the tag and the value is enclosed within double-quotes or single-quotes. The code title="elementTitle" is an example of en attribute; multiple attributes can be added to an element's beginning tag.

Boolean Attributes

Some attributes are Boolean, which means that merely adding the attribute name to the tag means the attribute is set. For example <input type="checkbox" checked> is equivelant to <input type="checkbox" checked="checked">. It is also acceptable to use empty quotes as the value for a boolean attribute; <input type="checkbox" checked=''>.
It is considered best-practice to use the long form (or redundant value) that specifies the full value of the boolean attribute especially if the attribute will be manipulated using JavaScript or jQuery.

HTML5 Global Attibute Reference

HTML5 provides a number of global attributes; these are attributes which may be applied to any HTML element.

  • accesskey Enables you to either specify a shortcut key to which to jump or to set focus to an element. As a rule, you shouldn’t use this because it can cause problems with other technologies.
  • class Used with CSS to specify one or more class names for an element.
  • contenteditable Specifies that the content within the tag can be edited.
  • contextmenu User can right-click an element to display a menu. At the time of this writing, no browser supports this attribute.
  • dir Enables you to specify left-to-right or right-to-left text direction for the content in an element.
  • draggable Specifies whether an element is draggable.
  • dropzone Enables you to specify the behavior of the dragged data when it’s dropped. Data can be copied, moved, or linked.
  • hidden Specifies that an element is not relevant.
  • id Specifies a unique id for an element.
  • lang Specifies the language (English, French, German, and so on) of the element’s content.
  • spellcheck Used with the lang attribute to enable you to indicate whether the element is to have its spelling and grammar checked.
  • style Specifies an inline CSS style for the element.
  • tabindex Sets the tabbing order of the element.
  • title Provides extra information about the element.
Working with void elements

Most HTML elements can have content. Elements are not required to have content (you can have an empty div for example) but there are certain elements called void elements that cannot have content. These are;

  • <area> Defies a hyperlink area with some text in an image map
  • <base> Specifies the document’s base URL or target for all relative URLs in the document
  • <br> Represents a line break
  • <col> Defies the properties of one or more columns within a <colgroup> element
  • <command> Defies a command that can be invoked by a user
  • <hr> Specifies a thematic change in content
  • <img> Defies an image
  • <input> Defies a typed data fild that allows the user to edit the data
  • <link> Defies a relationship between a document and an external resource such as a cascading style sheet
  • <keygen> Defies a key-pair generator control for forms that is used to encrypt data that will be passed to the server
  • <meta> Defies metadata that describes the HTML document
  • <param> Defies a parameter for an object
  • <source> Defies a multimedia resource for a <video> or <audio> element
  • <wbr> Optionally breaks up a large word at this element
Expando Attributes

Expando attributes are attributes that you define and are also known as author-defined attributes or custom attributes. These custom attributes can be used to attach data to an element. To ensure that the attribute name you choose does not conflict with existing, or future, attributes created by the W3C, it is important to prefix your attribute name with data-. For example, the following code <span data-custumernumber="123">CustomerName</span> allows us to add the customer number to the element.

Working with self-closing tags

Self-closing tags take the form <element /> where the closing slash is included in the opening tag. Any element that does not contains content may be represented as a self-closing tag, but for elements that can contain content (such as a p or a div) it is best-practice to use seperate opening and closing tags; e.g <div></div>. Empty elements that are capable of having content but currently don’t have content should have separate end tags.

NOTE: when using the <script src="…"></script> tag to reference an external JavaScript file, you must always include a seperate end tag.

Comments

Comments are not rendered on the page but are sent to the browser. Comments can be used to document code. Comments are added using the floowing syntax <!-- content sent to browser but not rendered in the page -->. It is not permissable to have a space between the <! and the two dashes in the opening tag. It is also not permissible to have three dashes in a row within the comment's content. Officially it is permissible to have a space between the two dashes and the > of the closing tag but in practice, this code <!-- comment -- > other page content will not close the comment and the other page content will not render in the page.

Conditional Comments

Internet Explorer recognises conditional comments in the form of <!--[if lte ie7]> <link href="styles-ie7.css" rel="stylesheet" /> <![endif]-->. This can be useful in delivering stylesheets to older, broken versions of Internet Explorer. In order to target browsers other than IE the following code can be used <!--[if !IE]> --> IE will not display this content, Firefox will <!-- <![endif]-->

Creating an HTML document

Every HTML document should have the following basic structure;

  • <!DOCTYPE html> - the doctype declaration
  • <html> - the root html element. This element contains;
    • <head> - the head element - this contains hidden information such as metadata and information about the document (i.e. the language) in addition to links to resources the socument requires.
    • <body> - the body element - this element contains the information to be displayed in the browser.

Lesson Summary

  • An element is composed of a starting tag, inner content, and an ending tag.
  • Browsers ignore tags that are not recognized.
  • HTML5 originates from HTML 4.01, not from XHTML.
  • The W3C is responsible for developing open standards for the web.
  • HTML elements provide structure, CSS style sheets provide presentation, and JavaScript provides behavior.
  • Use lowercase tag names.
  • Attribute values should always be quoted using either single quotes or double quotes.
  • Boolean attributes are attributes whose mere presence on the starting tag indicates that the option is set.
  • HTML5 defies global attributes, which are the set of attributes that can appear on any HTML5 element.
  • Self-closing tags are tags whose beginning and ending tags are together to create an element with no content. Self-closing tags should be used only with elements that cannot have content.
  • Void elements cannot have content. They should be created by using self-closing tags.
  • Expando attributes are attributes that you define and are also known as author-defined attributes or custom attributes. Prefix these attributes with “data-“.
  • You can use conditional comments to add a browser-specific source that will work with Internet Explorer but be treated as a comment by other browsers.
  • HTML entities are special characters and can be embedded in your HTML document by using the ampersand (&), the entity name, and a semicolon (;). You can also use the ampersand (&), the hash symbol (#), the entity number, and the semicolon (;).
  • Nonbreaking spaces can be used to render several contiguous spaces. You can also use nonbreaking spaces to keep two words from being separated by a line break.
  • The id attribute specifies a unique identifier for an element.
